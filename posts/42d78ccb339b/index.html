Objects in Parts & Articulation

有很多論文都是在解決有轉軸的模型生成，但是使用的方法不太一樣，有用3D diffusion model、也有用2D synthesis輔助的，也有用voxel的，可見拆件這個問題重要，尤其是轉軸，另外也表示這個問題目前非常開放，也非常困難，不是單純用agent可以解決的

關節，從照片來很難，還要知道如何活動、範圍等

FreeArt3D: Training-Free Articulated Object Generation using 3D Diffusion

有關節的資料很稀缺，那就不訓練了，直接使用 Trellis 3D 來當作pre-trained model來計算loss，Trellis 3D是可以用文字或圖片產生static 3D assets的model。

因此，整篇論文的大方向其實很簡單，就是拿使用者相機拍的照片當作是Trellis 3D的輸入，然後生成3D model，但不同的地方是，現在多了關節的參數，所以整個架構得要稍微重新設計。

具體來說，就是要透過Trellis中的 Flow Transformer 計算 SDS (Score Distillation Sampling) Loss ，用來引導3D Generation。

也就是說，為了讓Trellis計算loss，會先encode，然後進到Trellis中的 Flow Transformer 計算 SDS Loss ，告訴 Hash Grids 應該如何修改才能更像照片中的樣子，然後再經過decoder將其回體素空間，

https://microsoft.github.io/TRELLIS/

具體來說，無論是FreeArt3D或是Trellis，基本上都是使用volume data(voxels)來表示幾何，舉例來說，Trellis就是用離散的Occupancy Grid，但不同的是FreeArt3D則是使用連續的 Hash Grids ，應該是因為關節會有開合的連續變化，用一個連續的implcit表示方式使其優化比較穩定，因為我們會希望這個3D幾何動的時候應該要保持穩定，跟論文中的4D有關。

所以對照論文上的流程以及簡報得知，FreeArt3D一開始的幾何是隨機的Hash Grids，然後為了輸入給Trellis的模組，所以要先把Hash Grids以及關節參數利用Continuous Querying的機制轉換成Occupancy Grid，之後才能進到Stage 1的Trellis VAE Encoder中，然後再加上輸入的圖片一起輸入到 Flow Transformer ，此時我們並沒有將3D幾何做任何的rendering，而是相 Flow Transformer 做為一個Native 3D Diffusion Model，它理論上就具備理解 3D 空間結構的能力，因此不需要將 3D 轉回 2D 就能判斷這個幾何形狀是否正確，此時就可以計算出SDS loss，告訴我們應該如何修改才能更像照片中的樣子。

然後經過 Flow Transformer 的vector再經過Decoder，就會變回Occupancy field，此時可以再將與目前的Occupancy field計算 Voxel Loss ，然後因為Querying是連續可微的，所以都可以gradient descent更新 Hash Grids。 至於Stage 2以及後續細節這邊就不往下展開。

總之，要最佳化的東西就是三個可動部分的 Hash Grids、固定部分的 Hash Grids以及關節參數。幾何主要是用 Hash Grids 來implicit表示，然後為了具體化成可以理解的幾何，會將其轉換成類似voxels的Occupancy Grid，不同的地方是這個Grid是連續的，所以有一個Continuous Querying的機制。

比較反直覺的是，我們不需要相機參數，因為Trellis本身就是隨機一個角度就可以生成3d voxel的model了，但反之，我們給的照片也應該要符合Trellis訓練參數的樣態才行。另外，optimize之前要先給定好有幾個元件，同時，也要給定好關節型態，例如revolute、prismatic。

應用上，或許我們不在意外觀，或許可以直接用proxy，功能、模擬上是準確的，但外觀可能不是。

Generating Objects with Part-Articulation from a Single Image

本篇的整個流程稱之為DreamArt，要解決的問題與FreeArt3D類似，不過更加激進，試圖要透過一張圖就生成。

思路也很單純，就是現在的影片生成模型很強，那為甚麼不直接用來生成關節運動的圖片、影片呢？生成的結果就可以直接拿來引導3D Generation，因此我們就直接拿那些成熟的模型來用就好。那實際上，本篇是多個現有技術整合的流程。

基本上前面兩個階段都還算好理解，就是人給定照片跟可動部位的mask，然後透過Hunyuan3D、PartField之類的去生成完整的幾何。然後拿完整幾何去作出包含可動部分的完整圖片、mask去生成關節活動的影片。

最後，第三階段，透過可微分渲染優化運動參數，使 3D 模型運動與影片一致，這部分就是拿合成影片去優化關節參數，值得注意的是，因為這個流程輸入是一張圖片，所以比較沒有相機參數的問題，基本上直接在第一階段2D生成3D幾何的時候就會確定下來，之後第三階段基本上就是固定該視角去專注優化關節參數。

綜上所述，你可以發現整個流程用到很多現有的AI去補充不足的資料，這是好處也是壞處，因為只要其中有表現不好的，有限制的，基本上就會導致最終結果不好，就像木桶理論。

LARM: A Large Articulated Object Reconstruction Model

這邊的目標也是類似，想要解決的就是過去方法的兩個問題，希望能夠用少量的圖片就可以快速生成足夠精細的模型。

簡單來說，就是要訓練一個Model，可以直接inference就可以做到我們想要的結果，而這樣的方式你可以叫它Fast-Feedforward，因為就是跑了一次forward propagation就做完了任務，相對來說，另一種就是Optimization-based，就是它在runtime的時候需要花一些時間慢慢反覆優化才能夠做完，例如FreeArt3D。另一方面，前者就需要事前花大量時間與資料自己訓練model，後者則不用。

那本篇的概念也很簡單，就是將 Novel View Synthesis (NVS) 變擴產成 Novel View and Novel State Synthesis (NSS) ，因為現在有很多厲害的NVS，給定相機參數就可以生成特定視角的圖片，進一步來說，如果給定狀態的參數，是否也能生成特定狀態的圖片，例如抽屜關起來是0，打開是1，那給定0.5，我們就會預期能夠得到半開的抽屜圖片。

技術上來說，本篇可以視為是基於LVSM的擴展，將「關節狀態（joint state）」作為與相機位姿同等重要的輸入參數，因此許多訓練細節可以參考LVSM以及論文，這邊就不深入展開。

https://haian-jin.github.io/projects/LVSM/

那麼既然牽扯到訊練，就會有一個千古難題，就是帶關節的3D資料很少，所以訓練就分成兩階段，第一階段類似LVSM，就是透過讓model合成novel view來讓理解3D靜態物體，之後就是才是拿 PartNet-Mobility 中的資料去進一步訓練model，使其有合成novel state的能力。

那最後的問題就是，現在可以生成任意視角與狀態，那要怎麼重建？這部分就是透過Multi-view Synthesis以及讓LARM生成其他的maps，例如depth, mask等，然後透過 Shape As Points (SAP) 去重建出mesh，那這部分細節也就不展開了。

總而言之， LARM 本質上將 3D 重建視為「可動性感知 NVS（Novel View Synthesis）」的下游任務 。如果模型生成的虛擬視角、深度圖或遮罩不夠穩定或精確，最終的幾何重建確實會受到直接影響。另外，因為是需要訓練的方法，所以也會仰賴訓練資料，如果出現的訓練資料沒有的狀態變化，那自然也就會失敗。

ArtiLatent: Realistic Articulated 3D Object Generation via Structured Latents

本篇論文的概念將所有關節運動需要的屬性都直接放入voxel之中

整個inference的過程其實比較單純，可以直接看Stage 2，使用者給定圖片以及隨機的vector，然後經過Articulation-aware latent diffusion model生成Articulation-aware latent vector，概念上其實類似2D轉3D，因為這個model其實是拿TRELLIS來finetune而成，只是輸出不是voxels，而是一個latent vector，然後再經過Articulationa-ware VAE decoder生成帶有各種屬性的voxels。最後再透過Articulation-aware Gaussian Decoder把結果渲染出來。

所以訓練過程分成兩個階段，先要訓練出Articulationa-ware VAE decoder和Gaussian Decoder，然後再訓練出diffusion model，這部分細節就不展開。

Assembler: Scalable 3D Part Assembly via Anchor Point Diffusion

本篇想要解決的問題是零件的組裝問題，

解決辦法的概念也很簡單，輸入是多個部件的mesh，然後將其轉換成Anchor Points，其實就像是點雲，然後用Diffusion model去參考目標目標圖片，生成一個組合後的Anchor Points，然後再用 Least-squares fitting 去align 每個原件上去。 相對於傳統的方式則是用決定性的model去預測每個部件的transformation，但是因為組裝的可能性有多種，因此要生成一種loss最小的transformation變得困難，又transformation的表示本身較為複雜，其涵蓋旋轉位移，如果單純的用點雲來表示，基本就只用位移來表示，相對簡單，因為位移在空間中自然是連續的，且點雲本身也帶有形狀特徵。

那之後也展示了一些應用流程，像是又途中，你可以拿椅子的圖片去生成部件的圖片，然後經過3D生成，產生各部件的mesh，最後再透過Assembler將其組裝成完整的3D model。

LLM-Primitives: Large Language Model for 3D Reconstruction with Primitives

本篇概念跟上述的本質都不大相同，想要用限定好的primitive去組合出所有的model，這個概念我覺得有點類似 Constructive solid geometry ，就是認為所有物件都可以被拆成最簡單的單元。

如果依照這個前提，那要用多少元件組合，怎麼組合基本上就是一個不限長度的排列組合問題，一個極端的例子就是，我可以用無限多的球去逼近一個物體，就像point cloud一樣，而這樣的問題，其實也類似於接龍問題，所以自然的LLM理論上可以來幫忙。

技術上來說，輸入是3D點雲，然後經過3D 編碼器（Michelangelo）先將原始點雲 X 3 d ​ 提取為 3D 特徵 H 3 d ​，這些特徵包含了幾何的全局與局部資訊。 之後將這個3D embedding其與使用者的自然語言命令串接在一起，丟到Llama 2 7B中，當然，這個LLM是經過finetune的，不然他無法理解這個embedding。這個LLM會生成出json檔，詳細的文字說明每個primitive的transformation，進而組合成3D model。

有趣的是，Assembler不想用tranformation來組裝，因為有ambigous的問題，LLM-Primitives的方式則是用CSVR的表示方式來唯一化。

總結來說，具論文所述，重建出來的結果未必幾何上準確，但是使用者更喜歡，因為它有抓到人類喜歡的 語意 ，主要就是因為它是用文字來思考，而不仰仗任何視覺，最多就是用3D 編碼器。

Expressive and Structured Gaussian Representations

主要的概念就是希望可以讓3D GS可以容易的互動，而要能夠互動就得將其結構化，透過結構化的方式與其溝通，例如可以用feature去分類、用圖片去指引GS的變形，甚至feature本身就是neural的，調整GS的結構或表示方式也可以在效能上優化，將其調整成適合算法加速或是硬體加速的形式

Gradient-Weighted Feature Back-Projection: A Fast Alternative to Feature Distillation in 3D Gaussian Splatting

概念是上是希望在splat上能乘載更多feature，例如一些語意資料，大多數的方法(Feature 3DGS)提供2D圖片去訓練/最佳化 GS，但這篇地宣稱是可以利用pre-trained 3D GS即可，不須再耗費大量的時間做訓練。

實際上就是用其他常見的2D AI(DINOv2)去生成2D的標註資料，然後反投影回去，這樣的方法在point cloud也有出現過，當時的問題投影的位置勢必會有些誤差，造成一些點可能在不同視角有不連貫的結果，每個點的語意未必能夠清晰的辨別是0或1，但在GS的框架中，本身就有權重的概念，就可以利用它來去想辦法計算出每個點的feature，而他直接借用了訓練完畢的權重當作是feature的權重。

所以只要做 inference-time backpropagation ，去對每個點做加權平均即可。舉例來說，桌子與地板的交界處可能在某些視角模糊，此時該pixel應該會是模糊的由鄰近splat所構成，那這群點可能同時是桌子與地板，但可能在其他視角則可能可以比較清晰的區分出來，經過每個視角的加權平均，就可以知道每個splat的feature為何，當然feature也可以是更抽象的概念。

這篇最大的貢獻可能在於不需要重新訓練，但這是因為他借用了現成的3DGS的rendering用的權重，但若feature本身與rendering的效果較無關，那直接借用的權重就不再適合。

「既然你貢獻了這塊區域 30% 的影像，那你就直接接收這部分 30% 的顏色屬性。」這樣一來，操縱手不需要練習，只要戲演一次（掃描一次視角），每個人就都知道自己該具備什麼特徵了。

Rigidity-Aware 3D Gaussian Deformation from a Single Image

概念上很簡單，就如同標題寫的，如果我們有現成的3D GS，我們想要一張圖做為參考，去讓3D GS能夠變形，當然因為只有一張圖，所以有一些前提假設，也就是約束不足，例如該物體應該一定程度是剛體，才不會散掉產生比較合理的動畫。

所以會把物體區分成rigid parts再去做變形，那這個分群則是基於Gaussian-to-Pixel Matching，找到3D的點跟2D的對應關係

Neural Texture Splatting: Expressive 3D Gaussian Splatting for View Synthesis, Geometry, and Dynamic Reconstruction

簡單來說，就是覺得splat的上的texture用rgb表示過於粗淺，尤其是在較少view和動態重建上，用過於表層的feature難以去紀錄這些多變的現象，因為許多feature可能是更底層的物理性質，需要很多的rgb樣本才可以去表現。

例如比較簡單的reflection可能就是一個參數可以描述，但是如果只能用rgb來描述，那我們就需要夠多角度才能良好的表現正確的反射。

概念上來就是splat存的資訊會更抽象，且只有三個方向，而把這個資訊decode成rgb就要透過另一個NN來做，這是在differentiable rendering中再放入neural rendering。而最後的結果也很有趣，不只是渲染效果變好，連幾何都變好了，

理論上可以套用到其他的3D GS方法

JumpingGS: Level-jump 3D Gaussian Representation for Delicate Textures in Aerial Large-scale Scene Rendering

解決的問題是GS針對比較不起眼的細節，通常會用比較暴力的解決方式，就是用細節的平均來表示，主要是因為細節通常佔畫面比例不大，造成的loss有限，所以要馬出現一個很大的模糊，要馬就是針狀的splat，去滿足特定的區域，其他區域的loss太小就忽略。

解決方式就是用多層級的splat，然後設計一個機制去做level-jumping

TC-GS: A Faster Gaussian Splatting Module Utilizing Tensor Cores

主要是針對alpha blending的效能瓶頸想辦法用硬體做優化，簡單來說就是想辦法用矩陣運算去計算alpha blending，把GS的alpha blending拆成兩個tensor。

指此理論上已經結束，但是講者做了個快速的轉折講了實際的工程問題，就是tensor core是用FP16，主要是浮點數的精度問題，所以畫面看起來就被量化過，在離原點越遠的地方就會越不準，這個在CG的depth map之類的也會遇到。

所以解決方法也類似，就是將global的資訊normalize到local，也就是將值都拉到原點的附近計算( -8~+8)，計算完再transform回去。

講者很有趣

DeMapGS: Simultaneous Mesh Deformation and Surface Attribute Mapping via Gaussian Splatting

概念上就是把point-representation的GS，變成mesh，概念上的方法也很常見，就是用基本的mesh去變形，這種方式的好處就是可以自帶有良好的拓樸關係，反之，其表現力有限，因此會需用其他map輔助。

最佳化的過程會先做2DGS，也就是把每個點先當作surfel來變形mesh，因為我們希望mesh的幾何表面更穩定，然後再做3DGS，幫方向的資訊加進去，去補足一些幾何的視覺效果。

Advanced Representations and Rendering for 3D Scenes

其實主要也是3D GS，比較著重在3D GS本身的進階優化，這些優化都是比較通用的算法優化，讓GS可以更快、更好，例如更大量的同時渲染、更有效的分布、反射的更加準確(鏡子)等

LVT: Large-Scale Scene Reconstruction via Local View Transformers

講這個論文前，要先提到 GS-LRM （Large Reconstruction Model for 3D Gaussian Splatting），相較於傳統3DGS針對每個case來optimization，一種比較能夠泛化方式是有一個Transformer是已經有了一些相關3DGS的重建知識，可以讓直接Feed-forward model，只做一次Single forward pass就可以預測出整個場景的參數。

但GS-LRM重建3DGS的複雜度是隨著相機的數量指數增加，也就是每個相機都會兩兩配對計算，這會導致良好的重建需要大量的相機，同時會伴隨算力指數增加，這樣的概念叫做Global view transformer，那很明顯的是，論文要做的事情是Local View Transformers，也就是只當前相機只參考鄰近相機。

這樣的概念並不新穎，CNN就是用conv去取代FNN，那這個東西實際使用在transformer中就會長這樣

簡單來說，傳統上Tranformer的架構用在3DGS就是GVT，每個相機的參數會被轉換成K、Q、V，然後每個相機的K都去跟其他所有相機的Q計算再乘上V，LVT則是每個相機的K只跟鄰近相機的Q計算再乘上V。

所以接下來的問題是，那些算是鄰近相機，就像是conv要開多大一樣，這部分就要去看論文。

總之，結果上來說，相機只需考慮鄰近的相機參數，因為一定程度上無關的相機可能提供的資訊有很大的誤差，所以被排除掉的話可以有效的讓場景重建的更好更穩定，並且多考慮了View-Dependent Opacity，去把透明度的資訊明確的表示，讓頭髮這種遠看使用透明度來表示的東西能夠更好的表現。

只是比較有趣的是論文中有提到Relative Transformation Embeddings其中又有提到Plücker ray，這個以前沒有學過，但概念上應該不難，可能要找時間看看，以下是一些待讀的相關資料

CityGo: Lightweight Urban Modeling and Rendering with Proxy Buildings and Residual Gaussians

傳統方法有蠻多問題的，例如Traditional geometry-based approaches, such as Structure-from-Motion (SfM) [37, 39] and Multi-View Stereo (MVS)

3DGS的硬體要求高，尤其是VRAM

所以這篇做的概念不複雜，就是先產生一個粗模(Proxy Mesh)，然後將她與真實的差距表示成Residual Gaussians

換言之，就是大致的內容還是使用textured mesh來表示，因為還是比較省資源，然後細節，尤其是人可能比較在意的細節，在用GS去補充，而textured mesh的產生主要還是仰賴精細的3DGS重建，然後再拿精細的合成資料去產生，細節請看論文。

GSWT: Gaussian Splatting Wang Tiles

論文名稱很簡單，只有以前有稍微研究過texture synthesis, image analogy之類的應該都對於Wang Tiles不陌生，

概念上其實差不多，就是想辦法剪裁出一個可以無縫拼貼下去的patch單位，比較困難的是在於要做在3DGS上有一些實務的問題要解決，例如怎麼做LOD，怎麼避免一些跳動閃爍(popping)之類的artifact，還有後續的surface map，將這些合成出來的東西能夠貼到有起伏的地形上(height map)，或是貼到球體上之類的，詳細就要去看論文了

Spectral-GS: Taming 3D Gaussian Splatting with Spectral Entropy

主要就是要解決針狀的瑕疵，這樣的瑕疵其實前面的論文也有提到過，GS在面對這種局部的小細節問題，要馬splat像針，要馬就是整個模糊，因為這是最簡單降低loss的方法。

另一個像針的原因是出在rendering，尤其是渲染透明的東西，這個在傳統的頭髮、樹都會出現，遠近的渲染效果會因為透明度的範圍有所不同，所以這篇論文就針對這兩個部份去解決

細節一樣要看論文，但大致就是要有個機制去感知原始形狀，不能無腦的讓splats分裂或是變成針狀。

A compact stochastic representation for Monte Carlo Path Traced images

概念上十分有趣，就是面對傳統path tracing會遇到的雜訊問題，一般的方式就是平均，

如果把每個採樣的點都統計下來會發現其實她會形成一個分布，如果我們粗暴的平均會損失很多細節。

因此問題是，我們應該要建模出這些統計分布，只要我們有了這些分布，就能更有效地降躁，因為我們有了分布

因此概念上，論文是擁抱雜訊，因為雜訊才能表現出原始的分布，實際上就是在radial log space using Gaussian Mixture Models (GMMs)去做統計，細節請看論文。

Editable Physically-based Reflections in Raytraced Gaussian Radiance Fields

這篇論文要解決的問題就是反射，主要的問題在於3DGS雖然是view-dependent，但是基本上只與物體(splat)的出射角，也就是觀看角度有關而與入射角無關，因此物體可以encode不同視角觀看的結果，紀錄物體與相機之間的關係，但是物體與物體之間的關係則不會知道，例如鏡面反射，除非有對應的相機又拍到。

明顯的例子就是鏡子中的反射，那些反射應該是來自於盒子的背面，而不是鏡子本身的顏色，但是3DGS在沒有看到這個視角的情況下，就會錯把那些資訊encode到鏡子上，而盒子的背面因為沒有被相機直接看到，所以會失敗。

更準確來說，是因為GS會將所有的屬性都依附在geometry上，或者更直白的說，大部分的顏都會被當成diffuse處理，但實際上看G.T. (CG生成)會發現，specular比想像中多

因此，要做的事情很簡單，就是把diffuse跟specular單獨拉出來，然後計算loss，那至如何將specular拆出來，就仰賴現在成熟的decomposition network，再加上CG產生的場景就可以提供非常良好的訓練資料

但這邊要注意的是，因為我們要GS真的能夠學會物體之間的反射關係，所以在計算reflection的時候還是得有一套用在3DGS的path tracing機制(3DGRT)。

所以在訓練(最佳化)3DGS的時候會先用一個fine tune的SD2 model去將輸入buffer分解成一堆map，然後再拿這些map跟透過3DGRT渲染的結果(也可以分解成各種map)計算loss，去不停更新場景中splats的屬性。

而訓練結束後，每個splats上就有正確的diffuse跟specular，實際渲染的時候就直接用3DGRT去渲染即可，不需要跑SD。
