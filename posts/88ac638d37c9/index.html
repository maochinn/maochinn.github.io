Object-level Visual Prompts for Compositional Image Generation

論文又稱visual-composer，早上稍微遲到了3分鐘，所以開頭沒聽到

簡單來說，現在生成圖片常會使用IP-adaptor(Image Prompt Adapter)，用 Cross-Attention 來將文字和圖片當作指引序列中的KV，而目標影像（從雜訊開始）作為 Q，但這在實際上遇到了兩個問題，就是生成的身分常常會無法保存，以及concept bleeding和attribute leakage。

前者的問題來可以視為一個trade-off，就是如果要AI生成一個特徵很像特定人物的圖片，那就會傾向生成相似角度、相似外觀的結果，這就失去的彈性與創造力，但反之，如果要求生成比較有創造力的結果，可能就導致完全不向特定的人物，因此這個問題是身份保存（Identity Preservation）」與「多樣性（Diversity）的權衡。

本篇解決這個問題的方式是讓K、V有不同的顆粒度，想辦法讓K從來源中擷取比較抽象的全局特徵，例如Layout，V則是要能夠描述比較具體的局部特正，例如精確的外觀資訊，實際的運作就是某個像素Q去跟每個抽象的K做詢問，看看這個像素應該大概要放什麼樣的內容，例如左臉的右上角，而實際上要畫出左臉的右上角的時候，則要去參考對應K位置的V，但這個V了解特定人物的左臉右上角的細節，結果上就是K的vector長度較短，V的vector長度較長。

因此實作的方法是K會基於 Coarse Encoder，利用 CLIP 全域影像嵌入（Global image embedding）來提取鍵，V則會基於 Fine-Grained Encoder） ，提取 CLIP 影像編碼器倒數第二層的網格特徵（Grid features）作為值，簡單來說，本篇認為同一個embedding沒辦法同時描述全局跟局部的東西，所以人為的將他們用壓縮程度來區分。

至於前面提到的concept bleeding和attribute leakage，前者發生在某些模型（如 BLIP-Diffusion）在處理多個圖像時，會透過平均化輸入標記（Tokens）或將所有輸入視為單一統一提示來處理，這限制了模型區分與控制個別物件的能力，例如KOSMOS-G 在處理「紅色花瓶」與「黃色小鴨」時，生成了一個結合兩者特徵的混合物，這種現象通常發生在模型 無法區分不同的視覺提示（Visual Prompts） ，導致生成的物件在身份或特徵上發生 全局性的混淆或融合 。相對於全局的融合，attribute leakage更偏向於 局部特徵的錯誤轉移 。這通常發生在物件雖然被分開生成，但其中一個物件的局部特徵卻出現在另一個物件身上，例如，生成的「填充熊」可能會長出「小鴨的喙」，這是因為小鴨的屬性洩漏到了熊的區域。這通常與* *注意力機制（Attention Mechanism）* *的範圍有關。如果模型在生成某物件的區域時，其注意力圖（Attention Map）未能準確限制在該物件範圍內，就會抓取到其他視覺提示的特徵。

為了徹底解決您提到的 Attribute Leakage （屬性洩漏），VisualComposer 還引入了 構圖引導（Compositional Guidance）

• 在推理過程中，模型會透過分割技術（如 SAM2）將每個物件的注意力圖 限制在特定的區域內 。

• 這確保了當 Q 去 query 物件 A 的 K 時，不會不小心抓到物件 B 的 V，有效防止了例如「小鴨的特徵出現在熊身上」的洩漏問題，因為我們將V跟K用不同的embeding就會產生這個風險，但往下的細節我就不展開。

在圖像生成模型的架構中，我們可以從三個層次來理解 VisualComposer 的創新：

1. 基礎生成邏輯（Self-Attention）： 圖像生成的過程本質上是一種類似「接龍」的自回歸行為。 自注意力機制（Self-Attention） 讓模型在生成當前像素或特徵時，透過內部的 Query (Q) 與過去已生成的 Key (K) 進行匹配，並提取對應的 Value (V)。這確保了圖像內部的 語義連貫性 與 空間一致性 ，讓生成內容能「參考自己」來維持結構穩定。

2. 外部指引機制（Cross-Attention）： 為了讓生成過程符合使用者的意圖，模型引入了 交叉注意力機制（Cross-Attention） 。此時，Query (Q) 來自模型正在生成的圖像空間特徵，而 Key (K) 與 Value (V) 則來自外部的指引序列（如文字或影像提示）。模型透過 Q 去檢索指引序列中的 K，找出「應該在哪裡畫什麼」，再根據匹配結果融合 V 中的資訊。

3. 核心問題與解決方案（KV-Mixed Cross-Attention）： 傳統的適配器（如 IP-Adapter）對 K 與 V 使用相同的編碼來源，這會導致* *「身份保留」與「佈局多樣性」的權衡難題* *：若特徵太細節，模型會死板地模仿原圖姿勢（過度擬合）；若特徵太粗略，則會丟失物件特徵（身份模糊）。

4. VisualComposer 的核心創新在於將 K 與 V 的語義粒度進行 解耦（Decoupling） ：

◦ 佈局控制（Keys）： 使用 粗略編碼器（Coarse Encoder） ，透過窄瓶頸提取高階語義（如 CLIP 全域嵌入）。這提供了一個模糊的導引，讓模型在物件的姿勢與排列上有更多發揮空間，大幅提升了 生成的多樣性 。

◦ 外觀注入（Values）： 使用 細粒度編碼器（Fine-Grained Encoder） ，透過寬瓶頸提取細節特徵（如 CLIP 網格特徵）。這確保了在確定的佈局下，能填入極其精準的視覺細節，達成高品質的 身份保留 。

ComposeMe: Attribute-Specific Image Prompts for Controllable Human Image Generation

本偏要解決的是一個很特定具體的問題，就是想要用image prompt去指定圖片中人的髮型、臉容以及衣服，就像是IP-adaptor，剩下的資訊則是用文字來描述，主要應該還是因為人對於這三種Attribute特別敏感。

現有方法的侷限在於生成內容常常會像是複製貼上，尤其是人臉，這邊的觀察是之前之所以表現得不好是因為不能使用同個tokenizer(encoder)，因為每種encoder的特質不同，例如： Omni-ID （負責臉部身分保存）、 CLIP-L （負責髮型特徵）以及 SigLIP-L （負責服裝細節），其中因為 CLIP 比較在意紋理特徵，所以比較適合，最後將三個embedding相接起來送入 基於FLUX.1-dev的model中。值得注意的是，文字不像上述的tokens會直接串接起來，而是採用 Decoupled Cross-Attention ，簡單來說，就是各自計算完整的KQV，然後用Scaling factor來加權影響力，避免文字跟圖片的語意會混在一起。

訓練的時候，基本上就只是去訓練diffusion model後接的 Adapter(Decoupled Cross-Attention的模組) ，將將影像特徵（如臉部、髮型、服裝的標記）注入到模型中，比較值得注意的是，訓練的方式稱為Multi-Attribute Cross-Reference Training，就是訓練的時候就會刻意地讓不同的attribute用不同的排列組合出現，讓model可以學會不同attribute的語意該如何理解。

ComposeMe 技術架構與訓練邏輯概述

1. 現有方法的侷限與核心觀察 現有的個人化人像生成技術（如單一參考圖適配器）常導致生成結果出現生硬的「複製貼上感」，且無法對特定屬性進行精細的解耦控制。來源指出，這是因為傳統方法傾向於將整個人格視為單一概念，並使用統一的編碼器（Encoder），難以捕捉不同屬性的細微特徵。

2. 屬性特定分詞策略 (Attribute-Specific Tokenization) ComposeMe 提出針對臉部、髮型與服裝使用不同的 專屬分詞器 ，以發揮各自的特徵優勢：

• Omni-ID ：專門用於高保真地保存 臉部身分（Identity） 。

• CLIP-L ：實驗證明其對 髮型特徵 的表示最為有效。

• SigLIP-L ：能提供更豐富的 服裝細節 紋理。 系統會將這三種編碼後的嵌入（Embeddings）進行 串接（Concatenation） ，並加入位置嵌入（Positional Embeddings）以區分屬性，最後注入基於 FLUX.1-dev 的凍結模型中。

3. 解耦交叉注意力機制 (Decoupled Cross-Attention) 為了避免文字指令與影像特徵產生語意干擾，ComposeMe 採用了 解耦交叉注意力機制 。不同於影像 Tokens 的直接串接，文字與影像資訊會 各自計算獨立的 Key 與 Value 路徑 。推論時，透過* *縮放因子（Scaling factor, λ ）* *來加權調整影像參考圖對生成的影響力，確保模型在保留參考屬性的同時，能精確遵循文字要求的姿勢、表情或環境描述。

4. 多屬性跨參考訓練 (Multi-Attribute Cross-Reference Training) 訓練過程主要針對擴散模型後接的* *適配器（Adapter）* *模組進行優化。核心創新在於其訓練策略：

• 機制 ：訓練時會刻意將輸入的臉部、髮型與服裝屬性從 同一個人的不同照片 中分別提取，使其與目標影像之間處於「 故意不對齊（Misalignment） 」的狀態。

• 目的 ：這種排列組合強迫模型學會忽略原始參考圖中的背景、姿勢或表情等干擾因子，從而真正理解單一屬性的獨立語意，達成高品質的* *屬性解耦（Disentanglement）* *與自然融合

PartComposer: Learning and Composing Part-Level Concepts from Single-Image Examples

本偏要解決的其實就是想要準確的把圖片中的某個部位合成起來，這個概念比較類似的技術是Break-A-Scene，

https://github.com/google/break-a-scene

只不過這樣的技術被歸類為Subject-level的合成，意即合成的時候是以一個比較大的整體來合成，舉例來說，狗不會分成狗耳朵和狗鼻子，他們在訓練資料中往往是同時出現的。換言之Part-level就比較容易理解，例如是要單獨把物體的某個parts替換掉。

但現在的方法通常會需要多張圖片作為訓練資料，這就意味著需要大型數據及，同時如果超出的訓練集之外，也難以用one-shot的方式讓AI可以學會，另外，同時畫面中如果parts太多，也常常會開始出錯，有點類似前面提到的concept bleeding和attribute leakage，這邊稱之為Concept missing和Ambiguity。

推論流程很簡單，就是輸入 Compositional Prompt ，例如：”A photo of a partial chair composed of: <v2> , <v5> , <v7> on a clean white background” 就可以產生對應合成的照片，

但想當然AI要知道 <v2> , <v5> , <v7> 是甚麼要在訓練的時候告訴他，首先將這些 Placeholder Tokens 轉換成視覺特徵，然後再送入diffusion model中產生圖片。

訓練的時候需要訓練( Joint Fine-tuning ) encoder跟diffusin model，前者要確保Placeholder Tokens能夠正確地被text encoder embeding成正確的視覺特徵，後者則是生成圖片的同時，也會產生Cross-attention maps，確保生生成結果跟生成過程的parts空間分布一致， 這能強迫標記只影響其對應的影像區域，解決「概念纏結」問題，確保生成的結果在空間分佈上符合預期，避免不同零件的特徵混在一起。另外也會透過另一個CNN作為Concept Predictor去確保parts都有產生，且形狀正確。至於稀缺的資料方面，利用 Dynamic Data Synthesis 去排列組合，確保AI學習的時候能夠理解個別parts的邏輯。

簡單來說，面對 Concept missing、Ambiguity ，PartComposer的策略更加直接，就是用Concept Predictor，從結果去明確的強迫模型學到清清楚楚地標記出每個零件「是否存在（分類）」以及「在哪裡（分割）」。

PartComposer 的訓練流程採用兩階段優化策略，旨在從稀缺的單圖範例中學習精細的零件級概念：

1. 兩階段優化與特徵解耦： 訓練初期（第一階段）僅優化占位符標記（Placeholder Tokens）的嵌入向量，隨後（第二階段）則 聯合微調（Joint Fine-tuning）文字編碼器（Text Encoder）與注入 LoRA 的擴散模型（Diffusion U-Net） 。文字編碼器的微調確保標記能被精確編碼為具備身份識別度的視覺特徵；而 U-Net 則在生成過程中透過* *交叉注意力圖（Cross-attention maps）* *將零件標記與特定空間區域對齊，有效解決零件間的概念纏結（Entanglement）問題。

2. 基於資訊理論的雙重監督機制： 為了防止零件遺失或位置歧義，系統引入了一個名為 Concept Predictor 的 CNN 網路 ，藉由* *最大化互資訊（Mutual Information Maximization）* *來監督去噪後的潛在特徵（Latents）。該預測器包含兩個輸出頭：* *分類頭（Classification Head）* *負責檢查所有指定的零件是否確實存在，避免「零件遺失」；* *分割頭（Segmentation Head）* *則預測零件的精確定位與空間結構，確保零件的形態與佈局正確且不發生歧義。

3. 解決數據稀缺的動態合成管線： 面對單張影像導致的訓練資料不足，PartComposer 利用* *動態數據合成（Dynamic Data Synthesis）* *技術，將不同來源的零件進行隨機縮放、重疊與排列組合，產出多樣化的合成樣本。這種方式打破了原始影像中固定的結構先驗，強迫模型理解單個零件的獨立邏輯與重組能力，從而實現高品質的跨類別或類別內零件組合生成。

AGSwap: Overcoming Category Boundaries in Object Fusion via Adaptive Group Swapping

這篇的概念就比較單純，就是要混和兩個詞產生自然的圖片，然後方法也很直覺，就是把兩個詞變成embedding，然後隨機拼貼在一起變成swapping embedding然後丟到diffusion model，產生出來的結果用CLIP 餘弦相似度計算生成的融合影像 If ​ 與兩張原始影像之間的距離差，目標是這張圖不特別偏哪一邊，根據平衡分數的正負號，隨機翻轉交換向量 f 中的部分元素，動態調整混合權重，直到影像在兩個概念之間達到視覺平衡（閾值 ϵ =0.01）。

可以發現整個過程中不牽涉到任何訓練，只有使用不同model來更新embedding，而更新的方法 Adaptive Group Updating ，參考梯度下降法，根據更新過程中的振盪情況動態調整每次翻轉的向量元素數量（群組大小），以平衡收斂速度與穩定性，但我看起來跟Genetic Algorithm有點像，因為它處理的是 離散的二元向量 並使用* *隨機翻轉（突變）* *來尋找平衡點，它的行為模式在實作層面上確實更接近遺傳演算法中的隨機搜索與演化優化過程。

MALeR: Improving Compositional Fidelity in Layout-Guided Generation

使用者輸入畫面的text prompt和用bounding box表示的layout

那過去的問題在於，只要指令要求一多，就會混在一起，除了屬性之外，連位置資訊都是，所以這篇的目標就是

這篇也是不需要先行訓練的技術，主要就是設計好loss，讓diffusion model去做 Latent Optimization ，讓optimize 潛在變數(embedding)的時候能夠正確的denoise。

所以重點在於有甚麼loss term

  • Masked Latent Regularization 簡單來說要避免背景(非subject的框)中產生出subject，所以會有個Regularization去將背景區域的潛在變數錨定在更新前的值附近，防止其產生subject圖案
  • In-Distribution Latent Alignment 該機制透過計算 KL 散度（Kullback-Leibler Divergence），強制要求在去噪過程中的潛在變數必須與標準高斯分佈 N(0,1) 保持接近，這個部分沒有很理解，可能有需要再補相關的知識，總之概念上應該是optimization的時候用bbox去指引會造成一些分布上的偏離，所以要把他拉回來。
  • Subject-Attribute Association 就是cross-attension map的每個subject bbox中應該要含有對應subject的注意力，反之，也不能有其他subject的注意力，也就是正向跟反向的相似性loss。

但這樣的方式有其天生的侷限性，因為很多loss都是根據bounding box彼此獨立來計算，所以如果bounding box有重疊就可能會失敗。

這三篇論文（ VisualComposer 、 PartComposer 與 ComposeMe ）的核心共通問題，在於如何讓擴散模型在接收多個「視覺提示詞」（Visual Prompts）時，既能 準確保留參考對象的身份特徵（Identity Preservation） ，又能 靈活調整其在畫面中的佈局與姿勢（Compositional Diversity） 。

儘管問題相同，但它們基於不同的核心觀察，提出了各具特色的技術方案：

1. VisualComposer: 基於「KV 角色分工」的觀察

• 核心觀察： 研究者發現交叉注意力機制（Cross-Attention）中， Key（鍵）主要負責控制場景的佈局與空間分佈，而 Value（值）則決定細微的外觀特徵與身份細節 。

• 問題診斷： 使用單一編碼器會面臨「身份-多樣性權衡」：小容量編碼器能產生多樣佈局但損失細節；大容量編碼器保留了細節卻會「過擬合」原圖的姿勢。

• 解法： 提出 KV-Mixed 交叉注意力 。它使用一個「粗糙編碼器」來提取 Key（確保佈局靈活），並用一個「細緻編碼器」提取 Value（確保細節清晰），讓兩者在注意力層中結合，達到特徵與佈局的平衡。

2. PartComposer: 基於「資訊調節與數據稀缺」的觀察

• 核心觀察：

1. 現有方法缺乏對 Embedding 中所含資訊的 明確規範 ，導致在重組多個細微零件時，零件特徵容易纏結或遺失。

2. 在「單圖範例」的情況下，模型缺乏足夠的 零件排列組合邏輯 。

• 解法：

◦ 概念預測器（Concept Predictor）： 引入一個 CNN 網路來最大化「去噪潛在特徵」與「零件代碼」之間的 互資訊 。透過分類與分割任務，強制模型確保每個零件都存在且位置正確。

◦ 動態數據合成（Dynamic Data Synthesis）： 隨機排列組合零件來合成訓練樣本，讓模型理解每個零件是獨立的實體，而非死記硬背原圖結構。

3. ComposeMe: 基於「屬性特質與不對齊訓練」的觀察

• 核心觀察：

1. 人像的不同屬性（臉、髮型、衣服）具有不同的視覺特性， 單一編碼器無法同時完美捕捉這三者 。

2. 如果訓練時輸入與目標是同一張圖，模型會學會「複製貼上」而非「理解語意」，導致無法跟隨文字指令調整姿勢。

• 解法：

◦ 屬性特定分詞策略（Attribute-Specific Tokenization）： 針對不同屬性使用最適合的編碼器（如 Omni-ID 負責臉部、CLIP-L 負責頭髮、SigLIP-L 負責服裝）。

◦ 跨參考訓練（Multi-Attribute Cross-Reference Training）： 訓練時故意將「輸入零件」與「目標生成圖」進行 不對齊配對 （例如拿 A 照片的臉配 B 照片的衣），強迫模型學會解耦視覺特徵與原始姿勢。

3D SMoE Splatting for Edge-aware Realtime Radiance Field Rendering

本篇要解決的問題很單純，就是想讓3DGS的邊緣可以更有效的表示，因為傳統上的混和就是用高斯分布加權平均，而本篇則提出使用Steered Mixture of Experts( SMoE )，簡單來說，就是用 Softmax 去平均。

這邊稍微解釋一下 Steered Mixture of Experts ，概念是來自 Sparse Steered Mixture of-Experts (SMoE) regression networks for Image and Video compression ，首先，無論是SMoE或是3DGS都是屬於 回歸（Regression）的方法 ，其目標是透過一組參數化的函數，來 擬合或預測連續的數值信號 。

其次兩者其實都是由多個高斯核組成的，只是混合的方式高斯核的方式不同， SMoE 通常使用Weighted Sum of Soft-Gates，而3DGS則是Weighted Sum of Gaussians，兩者實際上都是使用高斯核。

https://github.com/bochinski/tf-smoe , https://en.wikipedia.org/wiki/Voronoi_diagram

從 SMoE 在影像壓縮的應用上，我們可以看到它用300個高斯核去還原一幅畫，那就可以大量壓縮所需的容量，而它的混合結果像是領土切割，甚至有點像是Voronoi Diagram。

而問題就是每個高斯核 中心的顏色位置 與 權重 該如何找到，而前者在 SMoE 稱為 專家 ，後者稱之為 門控函數 ，但無論3DGS或是 SMoE 都一樣用regression的方式來找，前者用gradient descent，後者用最大概似估計（Expectation-Maximization）。

因此，Sparse Steered Mixture of-Experts (SMoE) regression networks，其實就是用NN去做regression擬合找到多個稀疏的高斯核的參數，Steered這邊的意思是指這個高斯核會旋轉，所以3DGS其實只是用了比較簡單的加權平均的SMoE，所以本篇就是把這個混和方法重新帶回 3DGS 中，讓它變得更稀疏且更精確。

具體來說，其實就是在加權平均的時候把高斯核的可見度放到分母，如此一來，看不見的高斯核基本上就幾乎不影響顏色，反之，可見的高斯核的權重大幅上升，導致整片顏色趨於一致，以此邊緣自然就會凸顯出來。

Procedural Scene Programs for Open-Universe Scene Generation: LLM-Free Error Correction via Program Search

PSDL

生成場景的方法可以分成兩種，一個是Declarative Approach和Imperative Approach，前者代表性方法如 Holodeck 與 DeclBase ，讓 LLM 用領域特定語言（DSL）描述物體間的關係（如「在上面」、「相鄰」），再交由求解器（Solver）計算出最終位置，一定程度上，內建的Solver 可以解決一些不合理的錯誤，但相對的如果有特殊的相對關係沒有寫在Solver中，則無法生成(如螺旋排列)。

後者代表性方法如 LayoutGPT ，其核心是讓 LLM 直接預測物體的座標、大小與旋轉，所以理論上可以生成任意的場景，但缺點就是如果場景有不合理的錯誤，例如空間上的不合理碰撞，缺乏錯誤修正機制。

所以本篇基於Imperative Approach，發明了Procedural Scene Description Language (PSDL)，它是Python-embedded DSL，簡單來說，就是用python腳本來描述場景中物件的相對關係。

其特別之處在於，你可以把物件的相對關係給明確宣告，例如桌子間距，而不是每個桌子在object-level各自定義，也就是說，如果遇到錯誤發生，例如物件超出範圍或是重疊，此時錯誤的修正機制是去修改關係的數值，而不是物件的數值，這稱之為relation-level error correction。

而這個修正機制就是PSDL的solver，Iterative Local Search，簡單來說就是去修改relation的數值，例如間距從0.2改成0.1，可能最後一張桌子就不會超出範圍，而這樣的修改是 Iterative ，會一直隨機編輯，直到所有可選的隨機編輯 都無法讓損失函數進一步下降超過一個特定閾值（Threshold） 時，迭代就會停止。比較敏銳的人就會發現，此時會掉入Local Optimum，同時也可以發現，使用Python來描述場景只是結構化的手段，重點在於描述物件之間的相對關係，要將破碎的訊息用結構化的方式讓LLM容易輸入輸出。

總而言之，你甚至可以想像成它用python interpreter當成是solver，只是這樣明顯不夠，所以它還是用了另一個機制(Iterative Local Search)來修成腳本中可能的錯誤，這些錯誤是由宣告式場景常見的錯誤，但是延伸到複雜的場景時，可能物件的關係由很多規則所限制，例如A物件同時被B、C、D的位置所影響，那此時 LLM 能否將破碎的自然語言規則，轉化為具備結構化邏輯的 PSDL 程式碼 就是很大的問題，換言之，我們不該自己去編輯PSDL，因為我們可能會讓整個關係出現衝突與矛盾，而LLM的作用就是從這些規則中找到可行的邏輯。

因此，可預見的，面對規則十分複雜的場景，LLM未必有能力能找到簡單使用for-loop就能規範的場景，而本篇基本上的情境在於相信LLM生成的結果已有一定水準之下，再用簡單的Local Search去解決局部的規則問題，換言之， LLM 產出 PSDL 的品質決定了場景的「上限」（結構與創意），而程序搜尋則確保了場景的「下限」（物理合理性） 。

Hierarchical Neural Semantic Representation for 3D Semantic Correspondence

本篇主要要解決的就是3D shape的對應問題，例如桌腳對應人腳，這不只是幾何特徵，還包含語意特徵，概念類似Day 1的3D Reconstruction & Intelligent Geometry，本篇提出的是hierarchical neural semantic representation (HNSR)，簡單來說就是拿現有的3D foundation model(pre-trained 3D generative models)的先驗知識(3D priors)，他們輸出的特徵向量來作為幾何、語意特徵，使用既有的model好處就是無須學習。

能夠將任意兩個幾何最一對一的對應有很多具體的應用，例如只要對一個3D model做好segment, texture等，就可以得到其他物體對應的segment, texture。

那傳統方法基本上就是純粹基於shape的特徵來計算對應關係，比如果尖銳的桌腳對應尖銳的桌腳，這類的方法無論是傳統的手工還是learning-based都還是侷限在shape的特徵，對於左腳右腳這種基本上完全相同幾何特徵的東西就難以區分。

那隨著3D Generative model經過海量的3D data學習之後，這些model一定程度上有學到比較抽象的語意特徵，以此去生成各種結構，例如桌子通常有四隻腳之類的，每隻腳的功能是支撐平衡。

因此本篇提出的架構很好理解，就是把3D model 放到 Generative model 中，然後把其中比較深層的特徵拿出來當作是全局的語意特徵，比較淺層的特徵拿出來當作是局部的幾何特徵，例如， TRELLIS 模型提取decoder第 4 層作為語意特徵， 第 6、8、10 層 作為幾何特徵。

而在做一對一對應關係的時候就是Progressive Correspondence Matching，首先第一階段是Global Initialization，用低解析度的全局語義特徵計算餘弦相似度（Cosine Similarity），之後再做Local Refinement，使用多解析度的局部幾何特徵來迭代地精確化匹配位置。

這篇另一個貢獻就是可以使用任意的model作為backbone，因為基本上只是提取特徵。

StyleSculptor: Zero-Shot Style-Controllable 3D Asset Generation with Texture-Geometry Dual Guidance

本篇要做的事情就是給定任意風格的圖片(不在訓練集中，所以稱zero-shot)，然後讓3D Assets，的貼圖和幾何都依據那張圖片做風格變換。

對於現有的方法的核心觀察

  1. 2D Style Transfer 3D Generation 就是先對 2D 圖像進行風格轉換再生成 3D，這往往會扭曲 3D 結構，導致語義內容失真(2D圖片比較少在做紋理的轉換，猜測是2D比較沒有所謂紋理跟幾何之分)，可以稱之為 Transfer-then-Generate 。 Problem: Distorting 3D structure / geometry.
  2. 3D Generation + 3D Style Transfer 對已生成的 3D 模型進行風格轉換，通常只能修改紋理，無法調整幾何結構以符合風格需求，可以稱之為 Generate-then-Transfer。 Problem: Can not adapt the geometric structure.

以此可以發現，其實蠻大的問題在於幾何結構的風格化，前者會不當的扭曲，後者則無法調整。

所以本篇的概念上就是要在 去噪推論（Denoising Inference） 的過程中避免幾何被不當的扭曲，所以要在 Style Injection 的同時要做到 Content Preserve ，概念上來說，前者就是要將風格注入到目前的內容之中，但同時，我們仍然希望可以維持物件原本的物理結構，所以會做後者來維持，但要注意的是 Content Preserve 的時候並不會去參考content image，純粹的就是維持既有的生成結果，而整個module就是 Style-Disentangled Attention (SD-Attn)。

這個module的核心問題就是content leakage，因為風格化的資訊在注入的時候很容易會過度侵入，例如，當你想要將「積木風格」套用在「機器人」身上時，如果發生內容洩漏，生成的結果可能不只是長得像積木的機器人，而是直接變成了一堆跟機器人無關的積木物件，失去了原本機器人的外形特徵。

實際上來看，整體流程如左圖，本篇基本上基於 TRELLIS 的transformer，只是把self-attension層擴展成 Style-Disentangled Attention (SD-Attn) ，原本的這個layer做的事情是完善3D model的幾何結構一致性，但現在我們希望在編輯幾何的時候同時要參考風格影像，所以這一層就不能只是self-attension，換言之，如果只保留self-attension，意即只保留fcp那一條流程，那其實就是 TRELLIS。

換言之，本篇實際上的作法會另外獨立推論兩個特徵fs跟fe，而fcp其實是fc複製過去的，所以 Transformer 區塊總共有三個分支。首先，會單獨複製目前的內容特徵fc成fcp去計算 TRELLIS 原本的self-attension，將結果特徵向量保存起來，然後再拿內容特徵fc跟風格特徵fs去計算cross-attension得到新的特徵向量。

之後要將兩個特徵向量做feature fusion，舉例來說，假設特徵向量共5個channels，那就取出前者向量中表示結構的特徵(例如 1、2、5 通道)，再拿後者被風格引導的向量，從中取出表示表示風格的特徵(第 3、4 通道)，將其拼接起來就是新的內容特徵fc+1。隨後新的fc+1, fs+1, fe+1再各自跟對應的content image, style image, edge image做cross-attension，這部分就是 TRELLIS 既有的流程。

所以現在有兩個問題，為什麼特徵可以這麼隨便拼接？以及要怎麼選擇通道？前者的答案是，因為他們的一開始的輸入是由同個cubic noise開始，然後過程中經過的NN權重全部基於 TRELLIS ，不再經過訓練，所以他們的結果會在同個latent space。另一個問題的答案基於他們的insight

  1. 3D features channels can be divided into content and style-significant channels.
  2. Style information exhibits better global consistency than content

他們認為、假設特徵可以區分成兩種，一種表示內容、一種表示風格，而後者的一致性較高，換言之，它的 Variance 較低，基於此，本篇用另一個邊緣特徵fe作為篩選器，用其中個別通道的一致性來當作依據，如果一致性低於一個標準，那這些通道就都屬於風格 Style-significant ，反之，都屬於 Content-significant 。

因此隨著風格的通道採用變多，模型的幾何也會被風格影響(content leakage)。剩餘的部分，例如細化等就請直接參考原文。

StyleSculptor 的核心價值在於它將 3D 生成視為一個 風格與內容特徵動態融合 的過程。 LLM 或預訓練模型負責提供高品質的結構上限，而 SD-Attn 與 SGC 機則透過參數化的特徵過濾確保了風格轉換的精確性與物理合理性的下限

這篇論文中，「核心觀察」是設計「核心方法」的科學理論基礎，而「核心方法」則是為了解決觀察中發現的挑戰而提出的具體技術實現。兩者的關係可以歸納為* *「發現特性 — 解決衝突 — 精準控制」* *。

以下是核心觀察與核心方法之間的具體對應關係：

1. 從「特徵共存」觀察到「跨 3D 注意力」方法

• 核心觀察 ：預訓練的 3D 整流流模型（如 TRELLIS）具備強大的 3D 潛在特徵，這些特徵已經同時編碼了物件的 幾何佈局與紋理資訊 。然而，簡單的特徵融合在內容與風格語義差異巨大時，會產生 語義衝突 。

• 核心方法（跨 3D 注意力） ：為了利用這些特徵並解決衝突，作者提出了 Cross-3D Attention 。它將內容生成的自注意力層替換為內容與風格特徵之間的 跨注意力機制 ，讓兩者在生成過程中逐步融合，從而確保即使輸入的語義不相關，特徵空間也能保持穩定。

2. 從「方差統計」觀察到「風格解構特徵選擇」方法

• 核心觀察 ：作者發現 3D 風格資訊具有更強的全域一致性 。具體來說，風格資訊在 3D 特徵塊（patches）中的變化較小（方差小），而內容資訊則因位置而異，變化較大（方差大）。

• 核心方法（SDFS 策略） ：基於此統計特性，作者開發了 SDFS（風格解構特徵選擇） 。透過計算 3D-Var（3D 特徵方差） ，系統能自動識別並篩選出「風格顯著」的通道進行注入，並過濾掉會導致* *「內容洩漏（Content Leakage）」* *的語義通道。

3. 從「注入順序」觀察到「風格引導控制」方法

• 核心觀察 ：在實驗過程中，作者觀察到一個有趣的現象：隨著風格通道數量 K 的增加，模型會 先採納紋理風格，接著才採納幾何風格 。

• 核心方法（SGC 機制） ：利用這個觀察，作者設計了 SGC（風格引導控制） 。這讓使用者可以透過調節超參數 K 來精確控制風格化的強度，甚至實現 僅限幾何 或 僅限紋理 的獨立轉移，極大提升了創作的靈活性。

4. 從「特徵丟失」觀察到「內容保留機制」

• 核心觀察 ：單純使用跨注意力融合特徵有時會導致原始內容的細節（如眼睛、零件結構）因過度風格化而喪失。

• 核心方法（內容保留機制） ：在 SD-Attn 模組中，作者加入了一個分支，專門處理原始內容特徵的副本（ fcp ​）。對於非風格通道，系統會使用自注意力來補充這些內容特徵，確保生成的資產在風格化的同時，依然能保留嚴格的 內容忠實度

Fuse3D: Generating 3D Assets Controlled by Multi-Image Fusion

本篇也是基於 TRELLIS pre-trained model的免訓練框架，目標就是使用者只要輸入圖片，然後mask想要的部位的region就可以組合出3D model。

那麼動機很單純，就是2D圖片可以透過類似的方式來組成圖片，那3D的為什麼不行？目前的 3D 生成（如文字轉 3D）雖然能透過詳細描述增加局部控制，但缺乏精確性且容易產生歧義。此外，現有方法大多只能處理單一控制目標，無法同時利用多張圖像獨立控制 3D 資產的不同區域

所以他們的概念想法大致就是先將所有圖片變成一個特徵向量，然後讓這個源自於 2D 影像的特徵透過語意跟3D 體素建立聯繫，有了這些聯繫方式，就可以讓3D voxels有足夠的資訊往下優化。

前者稱之為多條件融合模組 (Multi-Condition Fusion Module, MCFM)，根據使用者定義的 2D 區域遮罩（Masks），篩選出對應的特徵標記（Tokens），並將其與全域摘要標記（如 [CLS] 和 [REG]）拼接成一組 統一的條件標記集（Unified Condition Tokens） 。

將2D影像的特徵處理完後，就可以透過 3D 語義感知對齊策略 (3D Semantic-Aware Alignment Strategy)來將TRELLIS flow transformer初步生成的3D model跟Unified Condition Tokens做crosse attention，得到attention map，就可以把每個voxel跟與其相關的圖片區域(token)給找出來。

有了上述的對應關係後，就可以做 局部注意力增強 (Local Attention Enhancement) ，主要功能是當多張圖像的控制特徵在同一個 3D 區域重疊或產生衝突時，透過調整增強強度 λ ，系統能加強特定局部特徵的影響力，同時確保不同組件之間能夠平滑過渡，避免邊界產生突兀的視覺斷層。整個流程如以下

In-2–4D: Inbetweening from Two Single-View Images to 4D Generation

這篇要做到的是輸入起始跟終點的兩幀，然後得到4D的補幀內容。

概念上就是先直接生成中間影片的內容，然後拿差異比較大的幾個video fragment出來當key frame，然後各自使用如 Era3D 等多視角擴散模型，再利用這些生成的影像來訓練一個初步的、靜態的 3DGS，然後把每個keyframe的多視角圖片串成多視角影片，其中用 特徵傳播（Feature Propagation），將關鍵幀的多視角自注意力特徵（self-attention features）傳播到該片段的所有幀中。 最後，有了多視角的引導影片後，接著將靜態 3DGS 轉化為動態 4D 模型。

以前對於self-attention transformer的機制其實沒有比較深入的理解，藉此機會整理一下QKV的理解。

對於文字接龍來說，會用當下生成的這個字的語意Q去query過去的所有字的語意K，如果某個特定字的語意K能夠對(匹配、對應、相似)上這個Q，就會拿特定字的語意V去特徵空間(被訓練過的)中搜尋最相關的字，當然，實際上應該是所有字的語意V都會一起送去特徵空間，只是會有權重，在比較極端的情況下，只有一個字的時候(或是當下這個字可能只跟過去生成的某個字有關)，那此時只有一個字能夠匹配，也只有他有權重，那就只有那個字的語意V去特徵空間搜尋最相關的字。

cross-attention的典型應用是翻譯，cross-attention的方式在開頭是一樣的會用當下生成的這個字的語意Q去query，例如我做中翻英，那當下的字就是下一個英文字，但是此時被query的所有字並不是過去所有生成的英文字，而是原始的中文字，去找下一個英文字跟中文字的最相關，也就是拿當前英文字的語意Q去跟所有原句中所有中文的語意K去算相似度，然後再乘上對應的中文字語意V，在預訓練過的特徵空間(中文字典)中搜尋，跟這些中文字最有關的字。

Attention = 動態加權讀取記憶

  • K = 記憶的索引方式
  • V = 記憶的內容
  • Q = 我現在想讀什麼

Self-attention： 記憶來自「自己這一句話」

Cross-attention： 記憶來自「另一個序列（如原文、影像、音訊）」
