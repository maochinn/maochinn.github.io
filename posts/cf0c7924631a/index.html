FLTK算是蠻老的一個C++…

FLTK

FLTK算是蠻老的一個C+ + GU工具集，當然也可以用GLUT或是比較新的GLFW，但是這兩者都算是相當輕量化，所以相對來說如果要設計一些GUI介面就會相對不直覺，甚至按鈕、打字輸入可能都要自己想辦法(寫過一次又醜又難用)，或是掛一些lib來用，例如我有用過的nanogui。另一種就是包山包海的Qt，它提供相當直覺的GUI設計，但相對來說就是很肥，而且很多東西它都封裝起來，要自己去寫GL的code相對變得非常麻煩。

所以這篇就介紹FLTK，它可以自己設計GUI，也提供相對輕量的設計，至少可以自己寫底層的東西。

首先，先到 FLTK 下載code，這邊就選擇1.3.5。

Select Latest Release 1.3.5

然後把source code載下來，哪個載點原則上都可以。

Download fltk-1.3.5-source

載下來解壓縮完應該會看到fltk-1.3.5的檔案，接下來就要cmake它，這邊你要有VS跟cmake。Source code就寫解壓縮出來的那個檔案，build出來的資料夾原則要在哪裡都可以。

source code: …/fltk-1.3.5

build the binaries: …/build

然後請先選擇 Configure。

Press Configure

然後跳出來的東西就依照自己的配置自己選吧，選完就按 Finish

Cmake Configure

如果沒有跳出甚麼錯，應該會看到 Configuring done ，接下來按 Generate

結束應該會看到Generating done，接下來應該會看到build完資料夾會出現FLTK.sln，把它用VS打開，並選ALL_BUILD建置，這邊看你要用Debug或是Release去建，對應到後來應用程式是用Debug或是Release，這邊我全都要！

開啟你要做的專案，在屬性裡面連結FLTK，注意這邊組態跟平台要對。

這邊有三個地方要設定。

組態屬性>VC+ + 目錄>Include 目錄

組態屬性>VC+ + 目錄>程式庫目錄

組態屬性>連結器>輸入>其他相依性

Include的目錄是要放一開始解壓縮的目錄 …/fltk-1.3.5 跟cmake完的目錄 …/build ，主要裡面的FL資料夾裡面的header，讓專案知道相關的header可以從這兩個位置找。

接下來就是要連接lib，也就是要連結到我們cmake完的.lib，這邊要注意如果你用debug就要用 …/lib/debug ，release就要用 …/lib/release 。

最後是告訴專案有哪些額外的lib的名子，這邊因為是用…/lib/debug，所以就是把該資料夾裡面的lib寫進去。

如果要用Release也要用類似的設定，如果設定正確，就可以開始編了。

直接下去編可能會遇到這個問題，這邊我不確定是哪的問題，圖中的 WIN32 應該要被定義，然後 include “win32.H” ，但是不知道為甚麼沒有被定義，所以會跑去Include windows沒有的header，這邊最簡單的解決方法就是把 WIN32 改成 _WIN32 ，因為這個有被定義。

這樣原則上就可以了跑了，以上。

這邊我有編好的 FLTK 1.3.5 (VS2019 x64)，裡面的檔案都已經整理好了，Include就是所有Header，Lib就是已經編好的lib。所以只要把Include 目錄改成…/FLTK_1.3.5/include，程式庫目錄改成 . . ./FLTK_1.3.5/lib/Debug，這樣就不用自己cmake了。
