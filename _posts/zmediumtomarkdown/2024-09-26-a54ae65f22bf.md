### 【Omniverse】學習筆記 — 00：Kit\-based Development

English Version: [【Omniverse】Development 101: Getting Started with Kit\-Based Development](https://medium.com/maochinn/omniverse-development-101-getting-started-with-kit-based-development-78f8915183be?source=your_stories_page-------------------------------------){:target="_blank"}

合先敘明，本系列文章主要會基於NVIDIA的線上課程去示範部分Omniverse的功能，因此文章就會直接以課程作為目錄的依據。

詳細的內容請直接到官網查看，這邊我就只附上官方課程連結，主要是因為我也沒有跟NV有合作，感覺直接搬他們課程內容出來不大對，所以我只會寫一些一點簡介跟注意事項，其餘的就自己去看官方的Course\(需要註冊帳號\)。
### 目錄
- Developing an Omniverse Kit\-Based Application
- Creating and Customizing an Omniverse Extension
- Develop, Customize, and Publish in Omniverse With Extensions


雖然Omniverse炒起很兇，但是翻一翻其實沒有什麼文章在講怎麼開發，尤其現在如果到官網下載會建議你不要使用舊版\(Omniverse Launcher\)，而是要使用Kit\-based development來開發


![圖一：Download Omniverse Page\. Kit\-App\-Template \(left top\), Omniverse Kit SDK \(right top\) and Omniverse Launcher \(bottom\)](/assets/a54ae65f22bf/1*DTcqH0SfpvxJyGyFZxBcBA.png)

圖一：Download Omniverse Page\. Kit\-App\-Template \(left top\), Omniverse Kit SDK \(right top\) and Omniverse Launcher \(bottom\)

這邊做個簡介，圖一左上方的連結是 [**kit\-app\-template**](https://github.com/NVIDIA-Omniverse/kit-app-template){:target="_blank"} ，右上方的連結是Kit Base Editor，下方的連結Omniverse Launcher，其中


> **Omniverse Kit** is the SDK for building **Omniverse** applications like **Create** and **View** \. It can also be used to develop your own **Omniverse** applications\. 





[**Overview \- Omniverse Kit 106\.1\.0 documentation**](https://docs.omniverse.nvidia.com/kit/docs/kit-manual/latest/guide/kit_overview.html){:target="_blank"} 
[_As mentioned above, Pixar's Hydra is used to interface between USD and RTX\._ docs\.omniverse\.nvidia\.com](https://docs.omniverse.nvidia.com/kit/docs/kit-manual/latest/guide/kit_overview.html){:target="_blank"}

簡而言之，我們需要透過名為 **Omniverse Kit** 這個SDK去開發 **Omniverse** 的Application，但是直接使用SDK過於硬核，因此有三種方式讓你去開發自己的application。

首先是 **Omniverse Launcher** ，基本上就是官方已經用Kit build好的軟體，使用者只要打開就好了，基本上是完全不需要寫code，你可以當作是那種官方release版本的application，你可以自己去裝一些extension進去。

接著是 [**Kit Base Editor**](https://catalog.ngc.nvidia.com/orgs/nvidia/teams/omniverse/resources/kit-sdk/files){:target="_blank"} ，基本上就是官方幫你用好SDK build好了一個名為 **Kit Base Editor** 的application，你可以把它就當作是最小的一個application的範本，因為application其實就是由一堆extension所組成的，因此官方先幫你幫你用基本的extensions拼出一個基本的application。

最後是 [**kit\-app\-template**](https://github.com/NVIDIA-Omniverse/kit-app-template){:target="_blank"} ，這個就是需要你自己去build出application的版本，但是官方還是有給幾個template，只需要照著指令就可以build出幾個不同的application，這裡頭也包含了 **Kit Base Editor，** 否則直接丟一包SDK使用者也不知道怎麼使用，而既然是自己build，那能作的變化也就更多，例如application是由哪些extension所組成的都可以自己決定。

補充一點，基本是NV只有提供SDK，IDE的部分官方都是用VS code


![圖二：Omniverse Launcher](/assets/a54ae65f22bf/1*mOdRdm17KAXONYWjHU7rdg.png)

圖二：Omniverse Launcher

但是尷尬的是，大部分教學，包含官方的文件幾乎都是基於Omniverse Launcher，一個官方說2024年底就會停止支援的軟體，因此，這篇算是做個濫觴，整理一下我自己學Kit\-based Development的過程。

首先，既然目標是開發，因此就直接從需要自己build的版本開始，也就是 [**kit\-app\-template**](https://github.com/NVIDIA-Omniverse/kit-app-template){:target="_blank"}


[![](https://opengraph.githubassets.com/986417dbc039236e347d8e6806d878f92f2420cdf13fc5593a69a141f828dbc2/NVIDIA-Omniverse/kit-app-template)](https://github.com/NVIDIA-Omniverse/kit-app-template){:target="_blank"}


打開之後會看見它提供了5種template，也就是你可以直接build出這5種application，換言之，你可以基於這5種template去變化，或者說從這5種template去學習Kit怎麼使用


![圖三](/assets/a54ae65f22bf/1*W8XhGSBHHyr0HprVnStywg.png)

### Developing an Omniverse Kit\-Based Application

基本上這個Course是基礎，會教你怎麼用kit\-app\-template去build出Kit Base Editor，然後在application中添加一個簡單的extension，特別的地方在於，相對於application中添加，這邊是在\.Kit檔中添加extension，然後再依據這個\.Kit檔去build出application。

[**Course Detail \| NVIDIA**](https://learn.nvidia.com/courses/course-detail?course_id=course-v1:DLI+S-OV-11+V1){:target="_blank"} 
[_About Course Objectives Topics Covered Course Outline Stay Informed Contact Us_ learn\.nvidia\.com](https://learn.nvidia.com/courses/course-detail?course_id=course-v1:DLI+S-OV-11+V1){:target="_blank"}
### Creating and Customizing an Omniverse Extension

這個Course介紹簡單的UI怎麼添加，包含怎麼把extension放到menu裡面，還有把其他command的功能添加到自己的extension裡面。

[**Course Detail \| NVIDIA**](https://learn.nvidia.com/courses/course-detail?course_id=course-v1:DLI+S-OV-16+V1){:target="_blank"} 
[_About Course Objectives Topics Covered Course Outline Stay Informed Contact Us_ learn\.nvidia\.com](https://learn.nvidia.com/courses/course-detail?course_id=course-v1:DLI+S-OV-16+V1){:target="_blank"}
### Develop, Customize, and Publish in Omniverse With Extensions

[https://learn\.nvidia\.com/courses/course\-detail?course\_id=course\-v1:DLI\+S\-OV\-02\+V1](https://learn.nvidia.com/courses/course-detail?course_id=course-v1:DLI+S-OV-02+V1){:target="_blank"}

大體上包含了前面Course的內容，但是還有教其他debug的方法，大致分成兩個部分，一個是針對\.usd檔案的理解，這是因為Omniverse Application基本上都是基於\.usd，所謂的usd是Universal Scene Description，它Pixar推廣的檔案格式，簡而言之就是一個通用的格是希望可以統一一堆不同的檔案格式，在不同軟體中不需要轉檔來轉檔去，可以學到怎麼用python去修改以及存取USD場景裡面的物件。

另一個部份是會做出來一個scatter物件的半成品extension，事實上這個extension是好幾個Course都會要用到的，只是官方後來把這個extension下架了，但是這個extension可以讓你自己寫一個類似的出來，這樣後面的Course都可以複用這個自己寫的extension。

[**Course \| NVIDIA**](https://learn.nvidia.com/courses/course?course_id=course-v1:DLI+S-OV-02+V1){:target="_blank"} 
[_Edit description_ learn\.nvidia\.com](https://learn.nvidia.com/courses/course?course_id=course-v1:DLI+S-OV-02+V1){:target="_blank"}

比較需要注意的是，這個Course是基於Omniverse Launcher的，但是官方有說可以用Kit Base Editor去取代，也就是可以用 **kit\-app\-template** 自己build Editor或是下載Kit SDK再build Editor來跟著教學做。


> _Where this course refers to the **Omniverse Code** application, you can instead use the **Kit Base Editor** application you created as part of this setup process\._ 





其中，如果使用 [kit\-app\-template](https://github.com/NVIDIA-Omniverse/kit-app-template){:target="_blank"} 發現Menu中沒有Extensions，記得在執行時加上\-d，長得像這樣
```typescript
.\repo.bat launch -d
```

另外，Course還是有一些小缺漏\(也有可能只是我自己沒注意到\)，但基本上有理解的話就知道要去改哪裡，例如，如果要建立一個command應該要記得在\_ \_init\_ \_ \.py裡面要改成
```python
__all__ = ["ScatterCreatePointInstancerCommand", "ScatterWindowExtension", "ScatterWindow"]

from .commands import ScatterCreatePointInstancerCommand
from .extension import ScatterWindowExtension
from .window import ScatterWindow
```

以及是要記得在window\.py中要uncomment
```python
ui.ComboBox(self._scatter_type_model)
```

才會將combox顯示出來。

這個Course因為是自己從0到1開發extensions，雖然大多數只是複製貼上，但是至少會對UI或是變數怎麼傳遞會比較有概念，當然，只是有概念而已，比較細節如function什麼時候呼叫、function中為甚麼要這樣定義等問題比較沒有著墨。舉例來說， `AbstractItemModel` ，有一些method去要定義，但是內容基本上就是照著寫，但是不知道為什麼要這樣寫
